Алгоритм генерации публичного и секретного ключа представляет из себя пять последовательных действий:
    1. Выбираются два различных случайных простых числа p и q заданного размера;
    2. Вычисляется их произведение n = p * q;
    3. Вычисляется значение функции Эйлера (f(n)) от числа n, то есть количество натуральных чисел, не превышающих n и взаимно простых с ним. F(n) = (p – 1) * (q – 1);

4. Выбирается число d (1 < d < f(n)), взаимно простое со значением функции f(n). Оно именуется секретной экспонентой. Обычно в качестве d берут простые числа, содержащие небольшое количество единичных бит в двоичной записи, например, простые из чисел Ферма: 17, 257 или 65537, так как в этом случае время, необходимое для шифрования с использованием быстрого возведения в степень, будет меньше;

5. Вычисляется число e, мультипликативно обратное к числу d по модулю f(n). Число e именуется открытой экспонентой; обычно оно вычисляется при помощи расширенного алгоритма Евклида;

После этого пара чисел (e, n) публикуется в качестве открытого ключа RSA, а пара (d, n) играет роль закрытого ключа и держится в секрете.

Такая схема имеет очень широкое прикладное применение и используется во многих сферах, примером служит защищенный протокол обмена данными между сервером и клиентом - https. При необходимости установки соединения клиент и сервер обмениваются публичными ключами, что позволяет организовать защищенный канал обмена данными.
